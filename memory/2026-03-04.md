## 2026-03-04 11:23 - Config Changes Being Implemented

### Source
Code review feedback from external agent analyzing CDP_Merged work.

### Problem Identified
Workflow degraded from Phase 1 (effective debugging) to Phase 2/3 (overclaiming). Core issue: calibration - too willing to expand scope and promote intention into completion.

### Config Changes

**1. Hard Completion Gate**
- Do not mark COMPLETE without passing validation in current session
- If test fails/times out/not run: status must be PARTIAL or BLOCKED

**2. Anti-Overclaim Rule**
- Every major claim needs concrete evidence: command, test, workflow ID, provider output, file diff
- No evidence attached = downgrade the claim

**3. Timeout = Blocked**
- Tests that hang/timeout are never "passing locally"
- Record as blocked verification

**4. Scope Expansion Restrictions**
- After urgent fix, do not open new top-level objectives
- Must be in latest handoff or highest-priority queue item
- Do not invent roadmap work during bugfix sessions

**5. One Task Class Per Session**
- Prefer one primary task class
- Do not mix app fix + infra build + automation + roadmap docs
- Exception: only if later step required by first

**6. Infrastructure Gate**
- Require local syntax/plan validation before claiming complete
- Require minimal smoke test before implementation complete

**7. Automation Gate**
- Recreated cron job ≠ fixed
- Require at least one post-change run succeeding without original failure mode

**8. Contradiction-First Rule**
- If docs/code/runtime disagree: stop new work
- Fix current-state docs before expanding scope

**9. Reporting Language Constraints**
- Avoid: "fixed," "complete," "verified," "all tests passing" without evidence
- Prefer: "implemented," "partially verified," "reported"
- Conservative bias: downgrade claim when in doubt

**10. Temperature Control**
- Lower temperature for implementation/status/handoffs
- Higher temperature only for explicitly requested brainstorming
- Default: skeptical, verification-focused

### Status
Paused all CDP_Merged changes. Implementing config changes in behavior.
