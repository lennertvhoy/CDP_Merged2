# UX/DX Audit Report - Critical Issues Requiring User Intervention

**Date:** 2026-03-08  
**Auditor:** Agent self-audit (prompted by user)  
**Scope:** Tools, services, and integrations where silent failures occur

---

## Executive Summary

Multiple critical UX issues exist where the system returns empty/incorrect results without clear explanation or fallback. These issues were **not proactively identified** until the user noticed and reported them.

**Severity: HIGH** - These issues damage trust in the system's reliability and require immediate fixes.

---

## Issues Found

### ✅ FIXED: Issue #1 - 360° Search Requires Exact Name Match

**Status:** Fixed in commit `ab3ee2d`

**Problem:** `search_companies_unified()` only searched `unified_company_360` view, which only includes companies with identity links. Query "Brouwerij Simon" returned 0 matches even though "Brouwerij Simon & Co" exists in Teamleader.

**Impact:** Users cannot find CRM/Exact companies until identity reconciliation is complete.

**Fix:** Enhanced search to query all three sources (unified view + direct CRM + direct Exact tables).

---

### ⚠️ OPEN: Issue #2 - Identity Links Empty = No Cross-Source 360° Views

**Status:** Documented, requires decision

**Problem:** `source_identity_links` table is empty (0 rows). Only 9 Exact customers are linked via the view. Teamleader's 57 companies are orphaned.

**Impact:** Even with the search fix, true "360°" cross-source views only work for 9 companies.

**Root Cause:** 
- Teamleader mock data has fake VAT numbers not matching real KBO records
- No identity reconciliation script has been run

**Options:**
1. Use the 9 linked Exact companies for demos (immediate)
2. Create matching mock KBO records for Teamleader companies (medium effort)
3. Run reconciliation and document limited results (transparent)

### 🔍 DISCOVERED: Issue #3 - Segment Export Fails Silently on Empty Results

**Location:** `src/ai_interface/tools/export.py:105-107`

**Problem:**
```python
result = await client.search_profiles(query, limit=limit)
if not result:
    raise RuntimeError("Failed to retrieve segment data.")
```

If Tracardi has no profiles for a segment, this raises a generic error instead of explaining the segment exists but has no members.

**Impact:** Users don't know if segment is broken or just empty.

**Fix Needed:** Check if segment exists but is empty vs. actual query failure. Return helpful message.

---

### 🔍 DISCOVERED: Issue #4 - Push to Resend Fails Without Explaining Why

**Location:** `src/ai_interface/tools/email.py:238-239`

**Problem:**
```python
if total_count == 0:
    return f"Segment '{segment_id}' contains no profiles to push."
```

This is better than #3 (returns message not error), but doesn't explain:
- Is the segment empty in PostgreSQL?
- Or are profiles missing from Tracardi?
- Or do profiles lack email addresses?

**Impact:** Users can't diagnose why their segment won't push.

**Fix Needed:** Return detailed breakdown: segment count, tracardi count, profiles with/without emails.

---

### 🔍 DISCOVERED: Issue #5 - PostgreSQL Search Returns Empty List Without Diagnostics

**Location:** `src/services/postgresql_search.py:136`

**Problem:** Returns `[]` (empty list) when query fails, with no indication of what went wrong.

**Impact:** API consumers can't distinguish between "no matches" and "query error".

**Fix Needed:** Return structured result with status, count, and error details.

---

### 🔍 DISCOVERED: Issue #6 - No Fuzzy Matching for Company Names in 360°

**Location:** `src/services/unified_360_queries.py` (search_companies_unified)

**Problem:** Uses `ILIKE '%query%'` which requires the search term to appear contiguously in the name. "Brouwerij Simon" won't match "Brouwerij Simon & Co" if the user types "Simon Brouwerij" or "Simon Co".

**Impact:** Users must know exact name order and spelling.

**Fix Needed:** 
1. Split query into words and search for each
2. Use PostgreSQL trigram similarity (already in `postgresql_client_optimized.py`)
3. Rank results by relevance

---

### 🔍 DISCOVERED: Issue #7 - No Fallback for Missing Segment in Tracardi

**Location:** `src/ai_interface/tools/search.py` (create_segment flow)

**Problem:** When creating a segment, if Tracardi projection fails, the PostgreSQL segment is created but user isn't informed about the mismatch.

**Impact:** Segment appears to work but activation to Tracardi destinations fails silently.

**Fix Needed:** Verify Tracardi segment creation and report both PostgreSQL and Tracardi counts.

---

### 🔍 DISCOVERED: Issue #8 - Identity Link Quality Tool Doesn't Check for Unlinked Records

**Location:** `src/ai_interface/tools/unified_360.py` (get_identity_link_quality)

**Problem:** Tool shows match rates but doesn't surface:
- Which specific CRM companies are unlinked
- Whether unlinked records have VAT numbers (linkable) or not
- Recommendations for manual linking

**Impact:** Operators can't take action to improve link quality.

**Fix Needed:** Return sample of unlinked records with linking potential assessment.

## Process Failures

### Why These Weren't Caught Earlier

1. **No systematic UX testing** - Only unit tests, no end-to-end user journey tests
2. **Mock data masked issues** - Tests used data that "just worked"
3. **No negative path testing** - Didn't test "what if data is missing/unlinked"
4. **Silent failures** - Issues returned empty results rather than errors

### Required Process Changes

1. **Add UX acceptance criteria** to all features:
   - "User can find company with partial name match"
   - "System explains why 360° view is incomplete"
   - "Empty results include diagnostic information"

2. **Create negative test suite**:
   - Search with non-existent names
   - Query unlinked companies
   - Export empty segments
   - Push segments with no emails

3. **Implement structured error responses**:
   ```json
   {
     "status": "incomplete",
     "result_count": 0,
     "reason": "no_identity_links",
     "suggestion": "Run identity reconciliation or use company VAT number",
     "data_sources_available": ["teamleader"],
     "data_sources_missing": ["kbo_link"]
   }
   ```

## User Communication Template

When presenting results, always include:

1. **What was found** (count + sample)
2. **What wasn't found** (and why)
3. **What can be done** (actionable next steps)
4. **Data completeness** (which sources contributed)

**Bad:**
> "No matches found"

**Good:**
> "Found 1 company in Teamleader matching 'Brouwerij Simon'. No KBO link exists, so this is a partial view. To see complete 360° profile: [Run identity reconciliation] [Search by VAT number] [View CRM data only]"
