# Obsolete

Files under `docs/obsolete/` are retained only for auditability.

Reasons a file belongs here:

- it contains unsafe historical details such as credentials, tokens, or secret-like values
- it gives misleading live-state or next-step instructions
- it duplicates current control docs in a way that increases operator confusion

Do not use these files for live operations, handoffs, or planning.

If a file is still needed as a technical reference, redact it and move it into `docs/archive/` or back into the active docs set.
