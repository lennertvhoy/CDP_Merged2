# Archive

Files under `docs/archive/` are preserved for context, traceability, and postmortem review.

They are **not** current sources of truth.

Use the repo root control docs for current state:

- `AGENTS.md`
- `STATUS.md`
- `PROJECT_STATE.yaml`
- `NEXT_ACTIONS.md`
- `BACKLOG.md`
- `WORKLOG.md`

Subfolders are organized by why the files are kept:

- `status_history/`: dated status snapshots and correction notes
- `planning/`: superseded plans or root-level planning copies
- `reports/`: implementation, cleanup, and investigation reports
- `operations/`: older SOPs and runbooks tied to legacy workflows
- `roadmaps/`: superseded architecture or delivery roadmaps
- `session_handoffs/`: old handoffs that may still explain prior decisions

If an archived file contains unsafe material, move it to `docs/obsolete/` or sanitize it.
